## 1. ゲーム概要

### 1.1 コンセプト

シノビガミの戦闘システム（プロット・間合い・忍法）を忠実に再現しつつ、
**キャラクター自作** と **チーム対戦** をブラウザ上で遊べる形にしたWebゲーム。

### 1.2 基本仕様

| 項目 | 仕様 |
|---|---|
| プレイ人数 | 2名以上 |
| チーム数 | 2〜N チーム（可変） |
| チーム構成 | 1チーム 1〜N人（可変） |
| 対戦形式 | 多チーム戦（A vs B、A vs B vs C、A vs B vs C vs D … など） |
| プレイ環境 | ブラウザ（PC/スマホ対応） |
| リアルタイム通信 | 必要（プロット同時公開のため） |

### 1.3 ゲームの流れ（大枠）

```
キャラクター作成 → チーム編成 → 対戦開始 → 戦闘（複数ラウンド） → 勝敗判定
```

## 2. キャラクター作成

### 2.1 流派

6大流派を採用。流派によって得意な特技分野が決まり、**得意分野の特技を使う判定は目標値-1**。

| 流派 | 読み | 得意分野 | 仇敵 |
|---|---|---|---|
| 斜歯忍軍 | はすばにんぐん | 器術 | 鞍馬神流 |
| 鞍馬神流 | くらましんりゅう | 体術 | 隠忍の血統 |
| ハグレモノ | はぐれもの | 忍術 | 斜歯忍軍 |
| 比良坂機関 | ひらさきかん | 謀術 | 私立御斎学園 |
| 私立御斎学園 | しりつおとぎがくえん | 戦術 | ハグレモノ |
| 隠忍の血統 | おにのけっとう | 妖術 | 比良坂機関 |

仇敵関係（輪になっている）：
```
斜歯忍軍 → 鞍馬神流 → 隠忍の血統 → 比良坂機関 → 私立御斎学園 → ハグレモノ → 斜歯忍軍
```

> ※ 流派はフレーバー・忍法選択の推奨として機能する（ハードロックはしない）

### 2.1.1 特技習得ルール（流派制約）

- キャラクターは特技を**合計6つ**習得する
- そのうち**最低3つは自分の流派の得意分野の特技**でなければならない
- 残り3つは任意の分野から自由に選べる

```
例）斜歯忍軍（得意：器術）のキャラクター
  器術から最低3つ選択必須：絡繰術・火術・針術（例）
  残り3つは自由：刀術（体術）・生存術（忍術）・医術（謀術）など
```

#### 【実装ロジック】代用判定の目標値計算

特技表は**2次元グリッド構造**（11行×6分野）。隣接する分野の列の間には**ギャップ列（空マス）が1列**ある。

```
列配置（実グリッド）:
器術  gap  体術  gap  忍術  gap  謀術  gap  戦術  gap  妖術
col0  col1 col2  col3 col4  col5 col6  col7 col8  col9 col10
```

分野インデックスと実グリッド列:

```
器術=0(col0)  体術=1(col2)  忍術=2(col4)  謀術=3(col6)  戦術=4(col8)  妖術=5(col10)
```

```
目標値 = 5 + |習得特技のrow - 指定特技のrow| + 有料ギャップ数

有料ギャップ数 = 経路上で通過するギャップ列のうち、無料でないものの数
※ 複数特技を習得している場合は合計距離が最も小さい特技を使う
```

**計算例（得意分野なし）：**
- 絡繰術(器術,row1)で掘削術(器術,row11)指定 → 縦|1-11|=10、ギャップ0 → **TV=15**
- 絡繰術(器術,row1)で騎乗術(体術,row1)指定 → 縦0、ギャップ1個 → **TV=6**

#### 【実装ロジック】得意分野のスキップ

得意分野の**両隣にあるギャップ列2本のコストが0**になる。

- 得意分野 fi=N のとき: ギャップ(fi=N-1 と fi=N の間) と ギャップ(fi=N と fi=N+1 の間) が無料

```
無料ギャップの判定:
ギャップ(fi=x と fi=x+1 の間) が無料
  ⟺ 得意分野インデックス === x  または  得意分野インデックス === x+1
```

**計算例（得意=忍術 fi=2、無料ギャップ: 体術-忍術間・忍術-謀術間）：**
- 地の利(戦術fi=4,row4) で 盗聴術(忍術fi=2,row4) 指定
  - 縦0、ギャップ: 謀術-忍術=無料・戦術-謀術=有料1個 → **TV=6**
- 走法(体術fi=1,row7) で 調査術(謀術fi=3,row4) 指定
  - 縦|7-4|=3、ギャップ: 体術-忍術=無料・忍術-謀術=無料 → **TV=8**

```javascript
// 実装（engine/core.js より）
function calcTargetValue(charSkillNames, requiredSkill, factionFavoredFieldIndex, skillsData) {
  if (!requiredSkill || requiredSkill === '自由') return 5;
  const reqEntry = skillsData.find(s => s.name === requiredSkill);
  if (!reqEntry) return 5;

  let minDist = Infinity;
  for (const skill of charSkillNames.map(n => skillsData.find(s => s.name === n)).filter(Boolean)) {
    const rowDist = Math.abs(skill.row - reqEntry.row);
    const loFI = Math.min(skill.field_index, reqEntry.field_index);
    const hiFI = Math.max(skill.field_index, reqEntry.field_index);
    let paidGaps = 0;
    for (let x = loFI; x < hiFI; x++) {
      if (factionFavoredFieldIndex !== x && factionFavoredFieldIndex !== x + 1) paidGaps++;
    }
    minDist = Math.min(minDist, rowDist + paidGaps);
  }
  return 5 + minDist;
}
```

### 2.3 忍法選択

キャラクターは **4〜6個の忍法** を選択してデッキを組む。

忍法の分類：

| 種別 | 説明 | タイミング |
|---|---|---|
| 攻撃忍法 | 相手に命中判定→回避判定をさせる | 自分の手番 |
| サポート忍法 | 判定への±修正・条件付き効果 | 任意（宣言制） |
| 装備忍法 | 戦闘中ずっと有効な常時効果 | 常時 |
| 奥義 | 1戦に1回だけ使える強力効果 | 特殊 |

- 基本値：**6点**
- 背景・流派によって増減（実装フェーズ2以降で対応）

## 3. 忍法データベース設計

### 3.1 忍法の2軸分類

忍法は**役割タイプ**と**習得制限**の2軸で分類される。

**軸① 役割タイプ（3種）**

| タイプ | 説明 |
|---|---|
| 攻撃忍法 | 主に攻撃に用いる |
| サポート忍法 | 特定のタイミングで発動させる |
| 装備忍法 | 常時発動するパッシブ |

**軸② 習得制限（2種）**

| 種別 | 説明 |
|---|---|
| 汎用忍法 | どのキャラクターでも習得可能 |
| 流派忍法 | 所属流派のキャラクターのみ習得可能 |

**組み合わせ：**
```
汎用・攻撃忍法
汎用・サポート忍法
汎用・装備忍法
流派・攻撃忍法
流派・サポート忍法
流派・装備忍法
```

### 3.2 忍法パラメータ定義

| # | パラメータ | 型 | 説明 |
|---|---|---|---|
| ① | name | string | 忍法名（漢字＋読み） |
| ② | type | enum | attack / support / equip |
| ③ | range | int | 間合（\|自プロット－相手プロット\| ≤ この値で対象選択可） |
| ④ | cost | int | コスト（0=なし。自プロット値以下の合計で支払う） |
| ⑤ | skill | string | 指定特技（分野名/特技名、"自由"の場合もあり） |
| ⑥ | effect | string | 効果テキスト |
| ⑦ | description | string | 概要（フレーバーテキスト） |
| ⑧ | is_taimahen | bool | 退魔編マークの有無 |
| - | multiple | bool | ※マーク=同名複数修得可否（trueで複数修得可） |
| - | availability | enum | general（汎用）/ faction（流派） |
| - | faction | string \| null | 流派忍法の場合は流派名、汎用はnull |

### 3.3 キャラクター作成における忍法習得ルール

- キャラクターは複数の忍法を習得できる（習得数は別途決定）
- **必ず1つ以上は自分の流派の流派忍法を習得**しなければならない
- 汎用忍法は流派を問わず何個でも選べる
- 同名の忍法は原則1つしか習得できない（※マークがある忍法は例外）

### 3.5 【実装ロジック】特殊効果の処理パターン

忍法の効果には以下のパターンがある。effectテキストだけでなく実装時に必要な状態管理を明記する。

| パターン | 該当忍法例 | 必要な状態管理 |
|---|---|---|
| 条件付き追加ダメージ（戦場条件） | 怪鳥・威光 | 戦場フラグ（高所など）・対象の属性 |
| 回避回数カウント | 凸拳 | 攻撃者×対象のペアごとに回避回数を保持、戦闘終了でリセット |
| 複数ダメージ種別 | 薙斬 | 命中判定時点での対象HP確認、接近戦＋射撃戦の順で処理 |
| 高ダメージ（コストあり） | 交叉 | 特になし（コスト消費のみ） |
| HP消費で変調回復 | 毒刎 | 自分のHPを1消費→変調リストをクリア |
| 攻撃回避時の再攻撃 | 連撃 | 回避イベントをトリガーに同忍法で再攻撃処理を発火 |
| ダメージ肩代わり | かばう | ダメージ発生イベントを割り込み、対象を自分に変更 |
| 命中判定前の割り込み | 火遁 | 命中判定前フェーズで発動、成功で攻撃者のファンブル値+1 |
| ラウンドカウント操作 | 閃軌 | ラウンド終了時に経過ラウンド数を+1追加加算 |
| 戦場変更 | 騎馬 | 戦場フラグをランダムに再抽選 |

| 生命力スロット追加 | 頑健 | HP_maxとスロット数を修得数に応じて計算（1個目+2、2個目以降+1） |
| ラウンド条件付き追加ダメージ | 飢渇 | 現在ラウンド数を参照し4以上で発動 |
| 集団戦命中判定ボーナス | 軍師 | 集団戦忍法使用時に命中判定+1を自動付与 |
| 奥義破り失敗カウント | 泪滴 | 奥義使用者ごとに失敗回数を保持、2回以上で自動成功フラグON |
| 忍具の代替使用 | 神通力 | 忍具システム実装時に対応（初期実装では後回し可） |

### 4.1 判定の基本

- **ダイス：** 2D6（6面体サイコロ2個の合計）
- **基本目標値：** 5以上で成功
- **代用判定：** 持っていない特技を使う場合、近い特技で代用。特技グリッド上の2D距離（縦距離＋有料ギャップ数）だけ目標値が上がる。詳細は2.2節参照
- **スペシャル：** 出目12 → 自動成功 ＋ 生命力1点回復
- **ファンブル：** 出目が自プロット値以下 → 自動失敗 ＋ 逆凪状態になる

### 4.1.1 逆凪（さかなぎ）

**発生条件：**
「攻撃の処理」または「ラウンドの終了」時にファンブル（出目 ≤ 自プロット値）を起こした場合に発生。

**効果：**
- 以降のあらゆる行為判定が**自動的に失敗**（達成値0扱い）
- 行為判定をともなわない行動（奥義の使用など）は問題なく使える

**解除条件：**
新しいラウンドになると自動解除。

**【実装ロジック】**
```
キャラクターの状態に is_sakanagiフラグを持たせる。
ファンブル発生時 → is_sakanagi = true
ラウンド終了時 → is_sakanagi = false（全キャラ一括リセット）
判定処理時 → is_sakanagi === true なら達成値0・失敗として処理
```

> ※ 逆凪はプロット高値のリスクとして機能する重要なルール。
> これがないと全員がプロット6を選ぶだけになりゲームが成立しない。

### 4.2 1ラウンドの処理フロー

```
① プロット前忍法フェーズ
   └─ 該当忍法を持つプレイヤーが宣言・発動（目標値F2）

② プロットフェーズ
   └─ 全員が1〜6を同時選択・公開
   └─ このプロット値がそのラウンドのファンブル値になる

③ 行動フェーズ（高プロット順）
   └─ 攻撃忍法宣言
   └─ コスト確認（使用忍法合計コスト ≤ 自プロット値）
   └─ 間合い確認（|自プロット − 相手プロット| ≤ 忍法の間合い）
   └─ 命中判定（2D6 ≥ 5）
   └─ 回避判定（相手が同特技で2D6 ≥ 5）
   └─ ダメージ処理（回避失敗時）

④ サポート忍法フェーズ
   └─ 命中判定・回避判定への修正（±1）を宣言

⑤ ラウンド終了フェーズ
   └─ ラウンド終了時忍法の処理
   └─ 脱落チェック

※ 同プロット値のキャラクターは同時処理（互いにダメージを与え合う）
```

### 4.3 生命力スロット

6分野それぞれに1スロット、合計6点。全スロットに印がついたら脱落。

```
[器術] ○
[体術] ○
[忍術] ○
[謀術] ○
[戦術] ○
[妖術] ○
```

### 4.4 ダメージ種別

**① 接近戦ダメージ**

```
① ランダムで分野を1つ決定
      ↓
② その分野のスロットが空き？
    YES → そのスロットに印（確定）
    NO  → 受けた側が空きスロットを自由に選んで印
```

基本はランダム強制。すでに埋まっている場合のみ受けた側に選択権が生まれる。

**② 射撃戦ダメージ**

```
受けた側が好きな分野を1つ選んで印
（常に自由選択）
```

**③ 集団戦ダメージ**

```
HPへのダメージなし。
変調表からランダムで変調を1つ受ける。
複数対象に同時に適用可能。
```

### 4.5 変調一覧

| 変調 | 効果 | 累積 |
|---|---|---|
| 敗走 | 忍具に異物・射出装置破損→忍具使用不可 | 不可 |
| マヒ | 身体の一部がマヒ→ランダムな特技が使用不可 | 可 |
| 重傷 | 大きな傷→命中・情報・感情判定などで接近戦ダメージ+1 | 可 |
| 行方不明 | 逃げて場面に登場できなくなる | 不可 |
| 忘却 | 頭が混乱→獲得している感情がランダムに変化 | 可 |
| 呪い | 呪術的効果→ランダムな忍法を修得していないものとして扱う | 可 |

### 4.6 脱落ルール

チーム戦における脱落条件：

- **生命力が0以下** になったキャラクターは脱落
- 脱落したキャラクターはそのラウンドから行動不能
- 次ラウンド以降のプロットにも参加しない

### 4.7 味方サポート（原作準拠）

- 自分のプロット以外のターンに、味方の判定に **±1の修正** を加えられる
- サポート忍法のコストは自プロット値内から支払う
- 1回の判定に複数の味方からサポートを受けられる（累積）

### 4.8 戦場

戦闘開始時、シーンプレイヤー（または該当ルールにより決定された者）が望むなら戦場を変更できる。
変更する場合は1D6を振り、戦場表からランダムに決定する。変更しない場合は「平地」（特に効果なし）で戦闘する。

#### 戦場表

| 出目 | 戦場 | 効果 |
|---|---|---|
| 1 | 平地 | 特になし |
| 2 | 水中 | 回避判定にマイナス2の修正 |
| 3 | 高所 | この戦場でファンブルすると接近戦ダメージを1点受ける |
| 4 | 悪天候 | すべての攻撃忍法の間合に1上昇 |

| 6 | 極地 | ラウンド終わりに1D6を振り、戦闘開始時からの経過ラウンド以下の目が出ると、戦闘参加者全員が接近戦ダメージ1点を受ける。この戦場から脱落した者は1D6を振り、変調の結果を適用する |

**【実装ロジック】**
```
戦場 = { id, name, effect_type, value }

  { id: "kosho", name: "高所", effect_type: "fumble_damage", value: 1 }
  { id: "suichu", name: "水中", effect_type: "dodge_penalty", value: -2 }

```

> ※「魔空」「騎馬」など戦場を変更する忍法は、この戦場表からランダム（または該当忍法の効果に応じて）再抽選する処理として実装する。

## 5. チーム戦ルール

### 5.1 チーム構成の柔軟性

- チーム数は **2以上、上限なし**（A・B・C・D … と任意に追加可能）
- 各チームの人数も **1人以上、上限なし**（チーム間で人数が異なっても良い）
- ロビーでホストがチーム数・チーム名を設定し、参加者が各チームに割り振られる

```
対戦パターン例：
  2チーム戦：A(3人) vs B(3人)
  3チーム戦：A(2人) vs B(2人) vs C(2人)
  4チーム戦：A(1人) vs B(2人) vs C(1人) vs D(3人)
  個人戦：  A(1人) vs B(1人) vs C(1人) vs D(1人)  ← チーム人数1人で疑似個人戦
```

### 5.2 行動順の決定

1. **全チーム・全キャラ** のプロット値を一覧化
2. 高いプロット値順に行動
3. 同プロット値は **同時処理**（互いの攻撃・回避を同時解決）

```
例）3チーム戦のプロット値：
  チームA: 鈴木(6) 田中(3) 佐藤(1)
  チームB: 山田(5) 伊藤(3) 中村(2)
  チームC: 服部(6) 霧隠(4)

行動順：鈴木(6)・服部(6)[同時] → 山田(5) → 霧隠(4) → 田中(3)・伊藤(3)[同時] → 中村(2) → 佐藤(1)
```

### 5.3 攻撃対象の選択

- **自チーム以外の誰でも** 攻撃対象に選べる（チームを跨いで自由に選択）
- 間合い条件（|自プロット − 相手プロット|）は **選んだ相手との差** で計算する
- 1ターンに攻撃できるのは **1体**（集団戦忍法を除く）

### 5.4 集団戦ダメージ（チーム戦拡張）

集団戦ダメージを持つ忍法を使った場合：
- **複数のキャラクター**（チームをまたいでも可）を対象に宣言できる
- それぞれに命中判定→回避判定を行う
- 全対象に間合い条件を満たす必要がある

### 5.5 チーム脱落

チーム内の **全メンバーが脱落** した時点でそのチームは敗退。
残りのチームで戦闘を継続する。

### 5.6 ラウンド数の設定

- 規定ラウンド数は **参加キャラクター総数** で決定する
- 例）3チーム合計9体 → 9ラウンド

## 6. 勝利条件

### 6.1 即時勝利（チーム脱落）

あるチームの **全メンバーの生命力が0** になった時点でそのチームは**敗退**。
最後の1チームが残った時点で即時勝利。

```
例）3チーム戦でチームBが全滅
  → チームBは敗退（2位以下確定）
  → チームA vs チームC の戦いが継続
  → チームCが全滅 → チームAが優勝
```

### 6.2 規定ラウンド終了時のランキング

全ラウンド終了時点で複数チームが生き残っている場合：

```
順位決定ルール：
  1. 残存生命力の合計（チーム内全生存キャラのHP合計）が多い順に順位付け
  2. 同点の場合は「脱落済みメンバーの最後に脱落したラウンドが遅い方が上位」
  3. それでも同点なら引き分け
```

### 6.3 結果表示

| 表示内容 | 詳細 |
|---|---|
| 最終順位 | 全チームの順位（1位〜Nチーム） |
| 残存HP | 各チームの残存HP合計と内訳 |
| 脱落順 | どのチームがどのラウンドで脱落したか |
| 戦闘ログ | ラウンドごとの行動履歴 |
